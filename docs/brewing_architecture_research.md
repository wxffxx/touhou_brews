# 典型酒精品类及其现实工艺归纳分析

为了在《东方酒艺》中建立一套既还原现实又具有游戏性的抽象机制，我们需要对现实中具有代表性的几类酒精饮品进行归类。

**酒精的本质来源是糖分的发酵**。因此，现实中的酿酒工艺主要是根据“糖分是如何获取的”以及“是否进行了提纯”来分类的：

## 1. 果酒类 (发酵糖类 - 直接发酵)
果实本身富含易于直接发酵的单糖/双糖，不需要“糖化”处理。
*   **代表饮品**：葡萄酒 (Wine)、苹果酒 (Cider)、蜂蜜酒 (Mead - 虽非水果但原理相同)。
*   **工艺流程**：`采摘 -> 压碎取汁 -> 加入酵母 -> 【发酵箱/桶】 -> 过滤装瓶 -> (可选) 橡木桶陈酿`。
*   **工艺特点**：流程最短，核心在于果实的品质和漫长的发酵及自然陈酿。

## 2. 谷物饮料/啤酒类 (发酵淀粉 - 单行糖化发酵)
谷物含有丰富的淀粉，但酵母无法直接利用淀粉，必须先将淀粉转化为糖（即“糖化”）。这类酒使用**谷物发芽**（产生淀粉酶）来完成糖化。
*   **代表饮品**：啤酒 (Beer)、艾尔 (Ale)、黑啤 (Stout)。
*   **工艺流程**：`谷物处理 -> 发芽(制麦 Malt) -> 烘焙 -> 糖化(与热水混合煮沸/Mash) -> 加啤酒花 -> 过滤得麦芽汁 -> 冷却加酵母 -> 【发酵桶】 -> 装瓶`。
*   **工艺特点**：前置处理繁琐，需要“发芽(Malting)”和“熬煮糖化(Mashing)”两个关键步骤，糖化和发酵是**分步进行**的。

## 3. 东方米酒类 (复杂发酵淀粉 - 并行复式发酵)
以中国黄酒和日本清酒为代表。同样是谷物（米），但东方不用发芽的方式，而是发明了**酒曲(Koji)**。米霉菌产生的酶一边把淀粉变成糖，酵母一边把生成的糖变成酒精。
*   **代表饮品**：清酒 (Sake)、米酒、黄酒。
*   **工艺流程**：`精米(研磨去壳) -> 蒸熟 -> 制曲(接种米曲霉/Koji) -> 混合(水+蒸米+米曲+酵母) -> 【发酵桶】(糖化与发酵同时进行) -> 压榨得生酒 -> 杀菌(巴氏) -> 窖藏`。
*   **工艺特点**：独有的**“制曲”**步骤，并且是在发酵桶中固体和液体混合着进行双重发酵。

## 4. 蒸馏酒 (烈酒 Spirits)
将上述任何一种经过发酵的低度数酒（果酒、啤酒、米酒均可），通过加热蒸馏的方式，提取高纯度酒精。
*   **代表饮品**：威士忌 (Whiskey - 啤酒蒸馏)、白兰地 (Brandy - 葡萄酒蒸馏)、伏特加 (Vodka - 薯类/谷物蒸馏)、白酒 (黄酒/高粱酒蒸馏)。
*   **工艺流程**：`发酵原酒 -> 【蒸馏器】(加热蒸发、冷凝) -> 高度数原浆 -> (可选)水割降度 / (可选)橡木桶长期陈放`。
*   **工艺特点**：必须经过**【蒸馏】**这一物理提纯步骤。

## 5. 浸泡/配制酒 (Liqueurs & Infusions)
使用已经酿造好的高度数烈酒作为基底，泡入水果、香料或药材，萃取其风味的再加工酒。
*   **代表饮品**：梅酒 (Umeshu - 传统是用伏特加或烧酒浸泡青梅)、苦艾酒、各类药酒。
*   **工艺流程**：`烈酒基底 + 加味材料(水果/花草) + 糖 -> 【泡酒罐/玻璃罐】密封静置 -> 过滤 -> 饮品`。
*   **工艺特点**：无需发酵，这是一个**物理萃取/浸泡**的过程。

## Minecraft 模组化抽象系统与方块设计 (设计蓝图)

通过以上归纳，我们可以把世界上的酒类生产高度抽象为以下 **5种核心机制方块/多方块结构**，形成一个具有完整逻辑闭环和可扩展性的科技树：

1.  **物理榨汁机/粉碎机 (Crusher & Presser)**：用于葡萄榨汁（果酒线）、压榨酒醪得清酒（米酒线）。
2.  **制麦/发芽培育槽 (Malting Tray & Koji Room)**：不需要电力，需要在特定光照和湿度下自然转变。用于制作麦芽（供啤酒）和米曲（供清酒）。
3.  **糖化煮锅/隔水蒸锅 (Mashing Kettle / Steamer)**：能在下方放置热源。用于把谷物+水熬煮成麦芽汁，或者将生米蒸熟。
4.  **大型发酵桶 (Fermentation Barrel)**：核心发酵容器（最好是多方块结构，类似沉浸工程的木桶）。能放入果汁/麦芽汁/米类混合物，随天数变成原酒。
5.  **蒸馏塔 (Distiller)**：中后期机器。输入低度酒+热量，产出高度烈酒。
6.  *额外项：窖藏橡木桶 & 小体积透明泡酒罐* (Aging Barrel & Infusion Jar)。
