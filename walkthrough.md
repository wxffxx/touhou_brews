# 完全重构的东方酒艺炼成系统 (NBT品质机制)

按照你的决策，我已经对系统底层的产出逻辑、成品数据结构以及视觉展示进行了**彻底的重写**。这个改动将核心玩法从“无脑自动化流水线”转变为了“硬核手工工艺酿酒”。

## 1. 动态品质引擎 & 自定义 NBT 物品 (`BrewItem`)

现在所有的酒不再是独立无聊的单例物品了。我创建了一个统御所有的 `BrewItem` 子类，它们内置了基于 NBT `{Quality: <0-5>}` 的动态解析能力：

* **名称自动覆盖**:
  * 0星: 强制叫作“失败的酒渣”，没有任何正面效果。
  * 1-4星: 会降格为“普通清酒”、“劣质青梅酒”等通用名。
  * 5星: 会显现出完整的东方对应传世名（如：蕾米莉亚的血红葡萄酒）。
* **自动药水时长乘算**:
  星级越高，同样喝下这瓶酒获得的增益时间就越久（甚至 5 星会解锁特殊的强力恢复效果，而 0 星喝下去会剧毒+反胃+饥饿）。
* **UI 动态文本六色光芒**:
  物品在工具提示中会按星级显示 0-5 个实心/空心星星 `★★★★★`。颜色按照设定的：灰色残渣/百/绿/蓝/紫/黄/橙，通过专门的 Hex 光谱算法实时渲染！

## 2. 占位符替换引擎 (ColorProvider 覆写)

所有的成品酒模型文件（JSON）已被强制映射为原版 `minecraft:item/potion_overlay`。
配合客户端植入的 `ColorProviderRegistry` 拦截器，它能在游戏运行时强行把你的酒染成：
* 樱花大吟酿：冰蓝色 `0xD6F8FF`
* 蕾米莉亚葡萄酒：血红色 `0x8A0303`
* 永琳的梅酒：荧光青绿 `0x98FF98`

## 3. 发酵桶重构 (Proof of Concept)

机器不再傻乎乎地读条就把东西扔在输出槽任由漏斗吸走了：
1. **禁用自动化**: 发酵桶的底层升级为 `WorldlyContainer`，硬核拦住了所有的漏斗行为。
2. **严苛的黄金比例**: 你要是直接放了一组（64个）材料进去，机器内部的 `QualityAlgorithm` 算法会直接判定“偏离 1:1 比例达到 63”。一瞬间给你扣成0分。
3. **时机掌控**: 机器内部的汁液开始默默流转（进度随着真实秒数上升）。你可以在UI界面看到第4个格子里一直飘着一个**“当前成品的实时预览图”**！
4. **截停出锅 (`extractBrew`)**: 你必须主动点击并从第4个格子里把那个预览物品“抢”出来，机器才会结算扣除发酵桶里的原材料。拿早了生涩，拿迟了发酸！

## 4. [新建] 创造模式 酿酒测试机 (`AdminBrewBlock`)

为了方便你测试所有颜色和种类的逻辑，我在“东方酒艺”创造栏目中增加了一个基岩隐形方块——`Creative Brew Tester`。
* **潜行 + 右键 (Shift+Click)**: 切换酒水种类（清酒 -> 红酒 -> 梅酒）
* **空手 + 右键 (Click)**: 切换当前的星级设定（0星 -> 1星 ... -> 5星循环）
* **直接左键/右击（非空手）**: 机器会立刻隔空吐出一瓶带有精准 NBT 品质星级的酒到你的手上！
