# 更新日志 (Changelog)

## [v1.1.0-alpha] - 动态品质系统与贴图重构

本次更新通过底层级的数据流重构和严格管控的材质自动化流水线，极大提高了《东方酒艺》(Touhou Brews) 模组的游玩深度和视觉质感。

### ✨ 核心机制重构 (Quality System Engine)
* **动态 NBT 品质星标（0-5星）**：舍弃了成品的静态化设定，所有的酒类输出都被收敛并封装为 `BrewItem`。
  * 引入了真实的 6色光谱体系 (白变灰、绿、蓝、紫、黄、橙)，用于渲染酒类的名称色阶。
  * 成品自带 0指5 颗实心星/空心星的动态工具提示 (Tooltip)。
* **动态配方验证与惩罚系数 (Quality Algorithm)**：
  * **禁用挂机自动化**：强制发酵桶 (`FermentationBarrel`) 使用 `WorldlyContainer` 阻塞底层漏斗的物流交互，回归硬核手工酿酒。
  * **配比算法介入**：机器不再被动吃特定数量的物品，而是吃下你的全部输入物品。如果由于你手滑丢入了夸张比例的原料（如 64个水稻 配 1杯水），偏离 1:1 黄金法则过高会遭致极其恶劣的惩罚分数，最终被强制炼成剧毒反胃的"失败残渣"。
  * **进度实时折损**：进度条满载 (100%) 时才为完美品质。拿取过早或是静置到发酵过度 (进度溢出) 会引发断崖式的品质暴跌。
* **增益强度波动**：根据你最终取得的 NBT 品质值，$1★$ 星只会发挥原本 $70\%$ 的药水时长，而完美的 $5★$ 星将提升至原本的 $150\%$ 并解锁额外的再生回复等传说特效。

### 🎨 画面质感提升 (Texture Pipeline Refactor)
* **确立绿色去底硬编码管线**：移除了极具腐蚀性的亮白色防锯齿抠图。通过纯绿幕 `#00FF00` 结合 Python OpenCV/Pillow 流水线 `texture_pipeline.py`，彻底杜绝了白米和白色背景的像素粘连问题，还原绝对锐利的 16x16 像素边缘。
* **玻璃块的特殊透光渲染**：
  * 完全修正并重画了泡酒罐 (`infusion_jar`) 和各种容器的顶部盖口，添加了铁丝卡扣与深邃的瓶体口径。
  * 赋予了深蓝色半透明像素过滤计算，开启 `RenderType.translucent()` 半透明剔除渲染后，从上帝视角往下看甚至由于截断了四围边角而呈现出了真实的圆柱体玻璃罐视差感，内部完全通透！
* **原版药水占位符 (Vanilla Placeholder Overlay)**：为所有受品质影响的新版酒种，使用底层拦截器 `ColorProviderRegistry` 覆写并继承 `minecraft:item/potion_overlay` 模型。实现了根据各自基底种类（如湛蓝的大吟酿、血红的蕾米莉亚红酒、荧光绿的梅酒）自动动态着色的极简视觉效果。

### ⚙️ 辅助开发组件
* **管理员虚空酿酒台 (`AdminBrewBlock`)**：为创造模式玩家新增了一个隐藏测试模块。将其放置在地表后，通过空手单击或潜行单击进行轮盘操作，它可以通过凭空干涉直接降临对应种类与确切星级的测试酒至玩家手中。

### 🐛 异常修复
* 修复了部分 1024x1024 粗糙贴图没有经过 Nearest-Neighbor 向下重采样强行压包导致内存爆库引发全屏幕所有草地和机器出现 “紫黑色残缺材质 (Missing Texture Checkerboard)” 的毁灭性渲染崩坏灾难。
